Week 1 Reflection

When I started this lab, my main goal was to understand how different platforms and tools connect in mobile and web app development. Each step introduced me to something new ‚Äî from version control to deployment and finally mobile app creation.

The first thing I did was set up my GitHub repository called ‚ÄúTemperatureConverterPWA‚Äù, and made the repository public so that my project could be shared and deployed later, and added a README file.

After creating my GitHub account, I cloned my repository using the command in the terminal. Next, I opened the project in Visual Studio Code. I installed Git and the GitHub extension so I could handle commits and pushes directly from VS Code. Using the terminal, I cloned my repository and started working on the project files.

Creating the Progressive Web App (PWA) was actually my favorite part. I designed the layout using HTML, added styling with CSS, and wrote JavaScript to convert temperatures between Celsius, Fahrenheit, and Kelvin. Then I added a manifest file to make the app installable and a service worker to enable offline functionality.

I deployed the GitHub page by selecting deploy from a branch to main and then to the root directory, and I saved it. After saving, GitHub generated a live URL for the app by itself.

After finishing the PWA, I deployed the same project on PythonAnywhere. At first, I made a mistake by putting the wrong directory path, which gave me a blank page. After correcting it, the app displayed perfectly. This step helped me understand how hosting works and how important correct file mapping is on cloud platforms.

Challenges

Throughout the process, I faced several challenges ‚Äî from Git synchronization issues to directory path errors and debugging JavaScript functions. However, every issue helped me learn something new about coding environments, deployment configurations, and mobile development tools.

Conclusion

This lab gave me practical experience in creating and deploying a cross-platform application using multiple technologies. By the end, I had not only built a working Temperature Converter App but also learned how to publish it online, run it on PythonAnywhere, and even test it on Android Studio. It was a complete learning experience that connected theory with hands-on practice and gave me the confidence to build and deploy my own apps in the future.

Week 4 Reflection ‚Äî Introduction to APIs

1. Which Storage, Browser, and Third-Party APIs did you choose, and why?

For the Storage API, I used LocalStorage to save and retrieve journal entries. I chose LocalStorage because it is simple to implement and allows data to persist even after the browser is closed or refreshed, which is ideal for a learning journal application.

For the Browser API, I used the Clipboard API and the Notifications API. The Clipboard API allows users to easily copy saved journal entries, improving usability. The Notifications API provides immediate feedback when an entry is successfully saved, which enhances the user experience.

For the Third-Party API, I integrated a public quote API to display a ‚ÄúDaily Inspiration‚Äù section. This was chosen to demonstrate how external data can be fetched and displayed dynamically, making the journal more engaging and interactive.

2. How did you integrate each API with DOM manipulation?

The Storage API was integrated by capturing form input values and saving them to LocalStorage as JavaScript objects. These stored entries are retrieved on page load and dynamically inserted into the DOM using JavaScript, allowing saved entries to be displayed without hard-coding content.

The Browser APIs interact with the DOM through event listeners. When a user clicks the ‚ÄúCopy‚Äù button, the Clipboard API copies the relevant text from the DOM. When an entry is saved, a custom event triggers the Notifications API to display a confirmation message.

The Third-Party API uses the fetch() method to retrieve quote data asynchronously. The response is processed and inserted into the DOM so that the quote updates dynamically when the page loads or when the user clicks the ‚ÄúNew Quote‚Äù button.

3. What challenges did you encounter, and how did you solve them?

One challenge I encountered was that the original quote API failed to load due to an HTTPS certificate error, which caused the browser to block the request. I identified this issue using the browser developer console. To solve this, I switched to a more reliable API and implemented a fallback system that displays local quotes if the external service fails.

Another challenge was resolving Git merge conflicts when pushing changes to GitHub. I resolved this by using the VS Code merge editor, carefully selecting the correct versions of files and committing the resolved changes. This helped me better understand version control workflows.

4. In what ways do these APIs improve your Learning Journal PWA?

These APIs significantly improve the functionality and usability of the Learning Journal PWA. LocalStorage ensures that journal entries are persistent and reliable. Browser APIs improve interactivity by providing quick actions such as copying text and receiving notifications. The Third-Party API adds dynamic external content, making the application more engaging.

Overall, the use of APIs transforms the journal from a static webpage into a more interactive and robust progressive web application that better supports reflective learning.

Week 6 Reflection ‚Äî Flask Backend Integration

1. Why is the frontend‚Äìbackend connection important?

The frontend‚Äìbackend connection is important because it allows the user interface to communicate with the server where data is stored and processed. The frontend handles user interaction and presentation, while the backend manages data storage, validation, and logic. By connecting the frontend to the backend using Fetch API, the application becomes dynamic and persistent, allowing user input to be saved and retrieved even after refreshing the page.

2. Which HTTP methods did you use in Flask, and why?

I used the GET, POST, and DELETE HTTP methods in Flask. The GET method was used to retrieve all reflections from the server. The POST method was used to submit new reflections from the frontend to the backend and save them in a JSON file. The DELETE method was added as an extra feature to remove reflections from the server. Each method was chosen because it follows RESTful conventions and clearly represents the action being performed.

3. What is the difference between using Flask to store and load JSON data and reading JSON directly in the browser?

Reading JSON directly in the browser only works for static data and does not allow secure or persistent updates. Using Flask to store and load JSON data allows the server to control how data is saved, validated, and accessed. Flask also enables multiple users to interact with the same data source and ensures that changes persist across sessions. This approach is more scalable and realistic for real-world applications.

4. Did you face any difficulties when running your project on PythonAnywhere? How did you handle them?

Yes, I faced several difficulties when running the project on PythonAnywhere, including routing errors, file path issues, and pages returning ‚ÄúNot Found‚Äù errors. I resolved these issues by creating proper Flask routes for each HTML page, organizing files into templates and static folders, and reloading the web app after each change. I also used error logs on PythonAnywhere to identify and fix backend issues.

5. What extra feature did you build into your PWA with Flask, and why did you add it?

I added a feature that allows users to delete reflections using a DELETE request handled by Flask. This feature improves usability by allowing users to manage their entries instead of only adding new ones. It also demonstrates a deeper understanding of RESTful APIs and backend interaction beyond the minimum requirements.

Week 7 Reflection ‚Äî Progressive Web App (PWA)

1. Why is it useful to enhance your Flask app with PWA features?

Enhancing my Flask app with Progressive Web App (PWA) features makes the application more reliable, accessible, and user-friendly. By adding PWA capabilities, the web app behaves more like a native application, allowing users to install it on their device and access it without relying entirely on a browser. This improves the overall user experience, especially on mobile devices. PWA features also improve performance and usability by enabling offline access and faster loading times, which is particularly useful when network connectivity is unstable.

2. What did you use to support offline access and dynamic data?

To support offline access, I used a service worker along with the Cache Storage API. The service worker intercepts network requests and serves cached files when the user is offline, ensuring that the core pages and assets remain accessible. For dynamic data, I used the Fetch API to retrieve reflections from the Flask backend in JSON format. When the app is online, fresh data is fetched from the server and stored in the cache so that previously loaded content can still be viewed when offline.

3. What extra feature did you add, and why?

As an extra feature, I added the ability to delete reflections from the journal. This feature improves usability by allowing users to manage their entries more effectively instead of being limited to adding content only. It also demonstrates interaction between the frontend and backend using HTTP methods beyond GET and POST, making the application more complete and practical.

4. Did you face any challenges deploying your PWA, and how did you solve them?

One of the main challenges I faced was configuring the service worker and ensuring that files were cached correctly without serving outdated content. This was resolved by clearing the browser cache and reloading the service worker through developer tools. Another issue involved missing PWA icons and file permission errors during deployment on PythonAnywhere. I solved this by correctly uploading the icon files to the static directory instead of trying to edit them. Testing the app using browser developer tools helped identify and fix these issues efficiently.
